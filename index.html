Project Idea

Overview

A new type of Eftpos machine or software upgrade for current Eftpos machines that uploads an itemised receipt list (csv file) to your bank when you make a purchase with your credit or debit card (including all type of payWave able devices).

Motivation

This project would put great power back in the customers hands, with paper receipts only lasting up to several months and the inconvenience of storing large piles of receipts, electronic receipts could greatly improve the customer experience. The project stems from the sms and email receipts that limited business do allow. The customer now would have the convivence of this service with the removed annoyance of entering personal details into business data bases or having to still personally organise electronic recepts.

Potential design ideas/overview

The Eftpos machine would be linked up to the point of sale system where when the Eftpos machine created an up link with the bank to verify payment transfer it would send across a compressed csv files from the POS to the bank and then would store the file on a secure file server inside the bank. Inside the server all files would be have a corresponding date and customer file number so that when the banks website server wanted to display the customers purchase file, it is able to search by customer number and then the website will either be able to display that data in multiple different way. Because all the different banks will store the file as a csv, every different bank will be able to use their own creativity and ingenuity to display the customers data in the best way possible. A banks website is outside the scope of this project’s ideas.

Tools and Technologies

Because of the personal nature of the information that will be used and manipulated in this project, security is one of the most important parts of this project. Because most modern Eftpos machine either use 4G or are connected by high speed internet connection bandwidth would not be an issue for security. All data sent up and down the link from the Eftpos machine to which ever bank server it travels to will need to be encrypted with AES256bit encryption. All banks would need to be brought together to agree on the same security protocols so that all different machines can upload data to their servers. The banks file servers would already have extensive security and security infrastructure, this would just need to be tested that it can handle the massive increase in data that they will receive with the csv files and that they have the appropriate hardware and software structure to store the data as well.

Skills and team required

A project of this size would require a large team of software engineers, business analytes, designers, network engineers and potentially many other professionals and skilled workers to complete these overall tasks. This project would have a large range of stakeholders as well, including the four big banks, the RBA, the federal government, retail companies listed on the ASX 200, the multiple companies that design and produce all types of Eftpos machine and many other smaller stakeholder who hold a valid interest in this project. A timeline for a project like this would be hard to pin down but could potential take multiple years to complete.

With such a large group of stakeholders, cooperation because a point that could create a lot of issues and potential be an obstacle its self.
